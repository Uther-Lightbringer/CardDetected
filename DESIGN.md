# CardDetect —— 联机 1v1 卡牌对战 · 核心玩法设计草案

> 版本 v0.1（2026-07-17）· 头脑风暴阶段产物
> 所有数值均为原型起点，需实测迭代调整。

## 1. 一句话定位

以「识破与欺骗」为核心的回合制 1v1 卡牌对战：
**盖放伪装打信息战，污染牌库打资源战，前后排站位打阵地战。**

- 对战形式：经典 1v1，回合制轮流行动
- 目标单局时长：8~12 分钟
- 目标平台：待定（原型阶段先做规则引擎，平台无关）

## 2. 三大核心机制（差异化钩子）

### 2.1 信息战：盖放 / 侦测 / 伪装

- **盖放（陷阱）**：陷阱牌以背面朝上形式进入「陷阱区」（每方最多 3 张），对手只能看到数量、看不到内容；满足触发条件时自动翻开结算。
- **潜伏（伪装单位）**：带「潜伏」关键词的单位可盖放打出，对外显示为无特征的 1/1「虚影」；在其发起攻击、被攻击或被指定时翻开，结算「翻开时」效果。
- **侦测**：查看对手的手牌 / 盖牌 / 牌库顶 N 张；部分侦测牌可**强制翻开**虚影——被强制翻开的单位不触发「翻开时」效果且当回合休眠（伪装被识破的惩罚）。

### 2.2 资源战：污染 / 净化

- **噪点**：1 费 1/1 无效果的白板近战单位牌。污染类效果将其洗入**对手**牌库，稀释对手抽牌质量。
- **洗牌干扰**：打乱对手牌库顺序，反制"牌库顶操控"类策略。
- **净化**：移除自己牌库中的噪点 / 使污染效果失效。
- 平衡原则：污染牌本身亏身材亏节奏，净化牌泛用性低——让"干扰"与"反干扰"形成动态博弈而非单边主流。

### 2.3 阵地战：前后两排战场

- 每方战场 = **前排 3 格 + 后排 3 格**，打出单位时选择部署位置。
- 攻击范围规则：
  - **近战**：只能攻击敌方前排单位；敌方前排为空时，才可攻击后排单位或玩家。
  - **远程**：可攻击任意目标（含玩家）。
  - **渗透**（关键词）：攻击时无视前排阻挡限制。
  - **护卫**（关键词）：敌方攻击时须优先攻击护卫单位（嘲讽）。
- 设计意图：前排是墙、后排是炮，阵型站位与数值同等重要。

## 3. 基础规则骨架

| 项目 | 规则 |
|---|---|
| 玩家血量 | 30 |
| 法力 | 每回合上限 +1（最高 10），回合开始回满 |
| 卡组 | 30 张，同名卡 ≤2，传说卡 ≤1 |
| 起始手牌 | 先手 3 张，后手 4 张 + 「先机芯片」（一次性：获得 1 点临时法力，或侦测对手 1 张手牌） |
| 抽牌 | 每回合开始抽 1 张；手牌上限 10 |
| 召唤失调 | 单位打出当回合不能攻击（「速攻」除外） |

**回合结构**
1. 开始阶段：法力 +1 回满，抽 1 张
2. 主要阶段：任意顺序打出卡牌
3. 战斗阶段：每个可就位单位攻击 1 次（目标须符合攻击范围规则）
4. 结束阶段：结算回合末效果

## 4. 卡牌类型

| 类型 | 说明 |
|---|---|
| 单位牌 | 攻/血 + 部署排 + 关键词；主要战斗手段 |
| 法术牌 | 打出即结算的一次性效果 |
| 陷阱牌 | 盖放进场，条件触发（信息战核心） |
| 强化牌 | 贴附在单位上的持续增益/减益（MVP 可并入法术） |

**关键词表（初版）**：近战 / 远程 / 潜伏 / 渗透 / 护卫 / 速攻 / 洞察（打出时侦测）/ 污染（打出时洗入噪点）/ 净化 / 战吼（打出时）/ 遗言（死亡时）

## 5. 数值框架（原型起点）

**标准身材表**（近战前排基准，无效果白板）：

| 费用 | 攻 | 血 | 备注 |
|---|---|---|---|
| 1 | 2 | 1 | 或 1/2 |
| 2 | 3 | 2 | 或 2/3 |
| 3 | 4 | 3 | |
| 4 | 4 | 5 | |
| 5 | 5 | 6 | |
| 6 | 6 | 7 | |
| 7+ | 7 | 8 | 高费曲线放缓，靠效果取胜 |

**修正规则**
- 后排远程：同费身材 −1 血（站位安全补偿）
- 正面关键词 ≈ 1 点身材（速攻 ≈ 1.5；潜伏 ≈ 0.5）

**效果价值换算**（粗略）
- 抽 1 张牌 ≈ 1.5 费
- 1 点直伤 ≈ 0.75 费（打脸价值上调）
- 侦测 1 张 ≈ 0.5 费
- 污染 1 张噪点 ≈ 0.5 费
- 强制翻开虚影 ≈ 1.5 费
- AOE 每点全场伤害 ≈ 1.5 费

## 6. 职业设计（4 个，机制人格）

| 职业 | 核心机制 | 风格 | 弱点 |
|---|---|---|---|
| 影袭者 | 潜伏、伪装、渗透、单点爆发 | 信息战刺客 | 前排薄弱，怕 AOE |
| 钢铁阵线 | 护卫、阵型增益、高质量前排 | 阵地推进 | 缺直伤，怕污染拖后期 |
| 噪点教团 | 污染、洗牌、烧牌库、利用噪点获益 | 资源消耗 | 怕净化，前期压制力一般 |
| 监察者 | 侦测、强制翻开、净化、反制陷阱 | 控制反制 | 节奏偏慢，进攻性弱 |

> MVP 阶段暂不做英雄技能，职业差异纯靠卡池；后续版本再补。

## 7. 联机技术要点（信息战机制的关键）

- **服务器权威**：客户端只发指令（出牌/攻击/结束回合），所有结算在服务器，天然防作弊。
- **按视角过滤状态**：盖牌、对手手牌、牌库顺序，服务器只向对应玩家下发其"可见"的信息——这是信息战类机制成立的技术前提。
- **指令 + 事件流**：客户端收到事件后播放表现动画；断线重连用状态快照恢复。

## 8. MVP 开发路线

- **阶段 0 · 规则引擎**：纯代码实现回合/法力/部署/战斗/三大机制结算，单元测试覆盖
- **阶段 1 · 热座对战**：本地双人轮流操作（极简 UI 或控制台），验证规则乐趣与单局时长
- **阶段 2 · 联机**：WebSocket 服务器权威 + 视角过滤 + 断线重连
- **阶段 3 · 匹配天梯 + 卡池扩充**：平衡性进入长期迭代
